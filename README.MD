# COYS Quiz: Tottenham Fan Test

## 프로젝트 소개

이 프로젝트는 오픈소스소프트웨어실습 중간고사 대체 과제로 제작한 Streamlit 기반 웹 애플리케이션입니다.

주제는 **토트넘 홋스퍼 팬 지식 퀴즈**입니다.  
사용자는 로그인 후 토트넘의 역사, 홈구장, 응원 구호, 상징, 라이벌 등과 관련된 퀴즈를 풀고, 최종 점수에 따라 팬 등급을 확인할 수 있습니다.

단순한 퀴즈가 아니라 축구 경기 흐름을 반영하여 `Kick-off`, `Match Result`, `Man of the Match` 등의 표현을 사용해 구성했습니다.

## 주요 기능

1. 첫 화면에 학번과 이름 표시  
2. 아이디/비밀번호 기반 로그인 기능  
3. `st.session_state`를 활용한 로그인 상태 유지  
4. `@st.cache_data`를 활용한 퀴즈 데이터 캐싱  
5. 토트넘 관련 객관식 퀴즈 풀이 기능  
6. 사용자의 선택과 정답을 비교한 점수 계산 기능  
7. 점수에 따른 팬 등급 결과 출력  
8. 전체 등급 체계 및 정답 확인 기능 제공

## 로그인 기능 설명

이 애플리케이션은 아이디와 비밀번호를 입력받는 로그인 기능을 포함하고 있습니다.

사용자가 입력한 아이디와 비밀번호를 미리 정의된 값과 비교하여 로그인 성공 여부를 판단합니다.  
아이디는 `spurs`, 비밀번호는 `1882`로 설정되어 있습니다.

로그인 상태는 Streamlit의 `st.session_state`를 활용하여 관리됩니다.  
Streamlit은 사용자 입력이 발생할 때마다 코드가 다시 실행되는 구조이기 때문에, 일반 변수로는 로그인 상태를 유지할 수 없습니다.

따라서 `session_state`를 사용하여 로그인 여부를 저장하고,  
로그인 전에는 로그인 화면만 표시되도록 하고, 로그인 후에는 퀴즈 화면이 표시되도록 구성했습니다.

이를 통해 실제 웹 애플리케이션과 유사한 로그인 흐름을 구현했습니다.

## 캐싱 기능 설명

퀴즈 데이터는 앱이 실행될 때마다 반복적으로 새로 생성할 필요가 없기 때문에 `@st.cache_data`를 사용했습니다.

Streamlit은 버튼 클릭이나 입력값 변경이 발생하면 코드가 다시 실행되는 특징이 있습니다.  
따라서 퀴즈 데이터를 불러오는 함수에 캐싱을 적용하여 한 번 불러온 데이터를 재사용하도록 했습니다.

이를 통해 불필요한 반복 생성을 줄이고, 앱 실행 흐름을 더 효율적으로 만들 수 있습니다.

## 퀴즈 결과 계산 방식

퀴즈는 총 10문제로 구성되어 있습니다.

각 문제마다 사용자가 선택한 답과 정답을 비교하고, 정답이면 점수를 1점씩 증가시킵니다.

점수에 따른 등급 체계는 다음과 같습니다.

| 점수 범위 | 등급 |
|---|---|
| 0~3점 | 입문 팬 |
| 4~7점 | 꽤 아는 스퍼스 팬 |
| 8~10점 | 진짜 COYS 팬 |

결과 화면에서는 사용자의 총점, 팬 등급, 전체 등급 체계, 정답 확인 내용을 함께 확인할 수 있습니다.
